HOW IS A WEBPAGE DOWNLOADED?

DOWNLOADING A WEBPAGE

When a user goes into a weg page, HTTP is utilised to download that page. Let’s follow through the process!

1. DIRECT BROWSER TO URL

When a URL(Uniform Resource Locator) is typed onto the search bar, ther “http” starting point indicates the browser that HTTP is used to fetch the document for the URL.

2. IP SEARCH

The Domain Name System then receives a message from the browser called a recursive DNS query. This message it sent for the DNS to translate the domain name into Internet Protocol addresses. The DNS server processes the query searching and returns the IP address to the user.

3. HTTP REQUEST

A HTTP request is then delivered when the browser recognises the IP address of the computer hosting the requested URL. The list of request can be seen in the network tab of the devtools wherea network log exhibits a list of request in a chronological order.

4. HTTP Response

A HTTP request is then delivered when the browser recognises the IP address of the computer hosting the requested URL. The list of request can be seen in the network tab of the devtools wherea network log exhibits a list of request in a chronological order.

Status line: contains the protocol version, status code, and the reason phrase. The status code is a three digit number. The first number in this code defines the type of response the web page is receiving. Here’s a categorisation of the Status Code:

  • 1xx Informational: Indicating the request has been received and the process is continuing.
  • 2xx Success: The response means that the request is not complied as valid authentication credentials for the resource is insufficient or invalid
  • 3xx Redirection: Involves more action to be done to fulfill the request
  • 4xx Client Error: Either the request has incorrect syntax or cannot be completed
  • 5xx Server Error: The server failed to fully execute the request.

Response Header Fields: this is additional information about the response which helps the browser to render the content. The content type can be seen as “text/javascript”. The content type informs the browser the kind of document it’s delivering back. Other examples of content-types are text/css, image/jpeg, text/html, and application/binary. The content-length indicates how many bytes are in the document for the browser to know how long the downloading process will be. When this process is finished, a HTTP response writes the document requested. When the browser has all the information needed, the document renders the requested content.
