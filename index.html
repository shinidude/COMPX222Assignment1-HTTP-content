WHAT IS HTTP AND HTTPS?

Let's explore Protocol first!

A protocol is a set of procedures that allows connection and data exchange between two places of a network. Protocol is used for the communication between the web-client, a web browser running on a device, and web server as they have to follow the same set of rules to be able to understand each other. There are different kinds of protocol such as FTP and Websocket, however in this site we will focus on HTTP and HTTPS.

HTTP and HTTPS

Hypertext transfer Protocol or HTTP is an application layer protocol made to exchange data from the web client and its server. There are web resource which is hosted by a web server. The web server could includes HTML, PDF, JSON, and etc. When a request of a resource is made from the web client, the HTTP allows access to the web server where the request will be processed.

After the process, the web server responses to the web-client with the resource. When the response is already given, the connection will be closed. This kind of transaction is recognise as a stateless protocol as the server will not retain any information from the last session.

Another version of HTTP called Hypertext Transfer Protocol Secure (HTTPS) is more secure. Because data communication between the web client and the web server is encrypted through an encryption protocol, Transport Layer Security (TLS), which guards against hackers. To do this, TLS utilises asymmetric public key infrastructure which involves two distinct keys:

  • Public key: it is securely accessible to anyone who communucates to the server.
  • Private key: it is stored in the web server and is controlled by the owner of the website. It is used to decrypt information encrypted by the public key.
